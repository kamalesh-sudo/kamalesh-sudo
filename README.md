## 🧭 Why DevSecOps + Automated QA?

> The security job market is crowded. Pentesting alone isn't a moat anymore.  
> But **engineers who can build secure pipelines AND break them** — that's rare.  
> I'm combining my backend dev skills and pentesting knowledge to become that engineer.

```
Current Market Reality         →    My Positioning
─────────────────────────────────────────────────────
"Junior Pentester" oversupply  →    DevSecOps Engineer (builds + secures)
Manual QA is dying             →    Automated QA with security baked in
Security as afterthought       →    Shift-Left Security in CI/CD pipelines
Tools knowledge only           →    Tools + code + architecture understanding
```
